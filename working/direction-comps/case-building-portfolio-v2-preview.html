Building This Portfolio

Senior Product Designer
Personal Practice
Mar – Apr 2026
Web (Static HTML/CSS/JS)

Challenge

I needed a portfolio that captures depth of craft while staying easy to scan and feeling authentically mine. The narrative was rich but scattered—and the industry was shifting under it.

The harder problem: every audience evaluates differently. Some roles weight AI workflow evidence, others want research methodology, others care about systems thinking or craft. No way to measure whether materials were landing across all of those criteria at once.

The evaluation gap—multiple audiences with different hiring criteria, no single tool to benchmark materials against all of them at once

Solution

I designed a practice infrastructure that produces, evaluates, and maintains the portfolio—sense-making tools to cut through bulk information, automations for recurring operational work, and repeatable skills that learn over time and interconnect.

The system handles prep, outreach, tracking, and quality checks so my time goes to craft and message.

System architecture—how 9 custom skills, living documents, and shared data hubs connect (interview-prep → tracker → overview → calendar → cheat sheets)

01 Building the Feedback Loop

Before writing a single case study, I needed systems that could tell me whether materials were working—and keep telling me as requirements changed. Every tool in this section runs against live data, not a one-time audit.

Competitive Benchmarking

I needed a baseline for the level I was targeting. I profiled 21 designers across 17 dimensions and identified three strategic archetypes—revealing where the field clusters and where the gaps are.

Competitive benchmark—21 designers scored across archetypes, showing cluster patterns and positioning gaps

Recruiter Panel

Self-assessment doesn’t scale across roles with different criteria. The solution: a three-stage evaluation pipeline scoring materials against real, current job postings—14 companies evaluated. Each run surfaces the specific gap between what a role needs and what the material delivers.

Recruiter panel output—per-company scores with gap analysis, showing what each role values and where materials fall short

Progressive Disclosure Framework

Content was duplicating across the resume, portfolio, deck, and LinkedIn. I formalized four layers, each with one job—scope, frameworks, decisions, process. No layer repeats what another says—they build on each other.

Progressive disclosure layers—LinkedIn → Resume → Portfolio → Deck, showing what content lives where and why

Screenshot Organization

1,576 screenshots across 7 projects, no organizing structure. The sorter I created uses a taxonomy that learns—each project sorted makes the next one faster and more accurate.

Screenshot sorter—before (flat folder of 1,576 images) vs. after (organized surface taxonomy with contact sheets)

02 Removing Operational Friction

Every hour spent on logistics—prepping for calls, tracking contacts, re-establishing context—was an hour not spent on the portfolio itself. I identified the repeating workflows and turned each one into a system.

Interview Prep System

Each recruiter call meant manually researching the company, re-reading my own materials, and assembling notes from scratch. Now a single skill generates cheat sheets from live research and the tracker, links them in calendar events, and syncs debrief notes back after the call. One input, everything updates.

Interview prep flow—calendar event detected → cheat sheet generated → linked in calendar → debrief after call → tracker updated → overview regenerated

Recruiter Tracker

Contacts, stages, and company intel were scattered across emails, messages, and notes. I consolidated into a single source of truth—32 leads, 24 columns, five sheets. Every skill reads from and writes to this file.

Tracker architecture—5-sheet structure showing how data flows in from skills and out to living documents

Persistent Memory

Context was lost between sessions—every conversation started from scratch. I designed a four-file memory architecture that self-heals—an automated script validates from the filesystem, flags stale content, and detects drift. Weeks of work stay continuous.

Memory architecture—four files, each with one job, showing how session continuity works across weeks of collaboration

Outreach Automation

Drafting outreach meant re-reading job postings and manually tailoring each message. An automated engine now generates targeted emails and LinkedIn messages with progressive disclosure, links prep materials via the calendar, and stages follow-up drafts after calls.

Outreach pipeline—company intel → drafted email → calendar detection → follow-up staging

03 The Collaboration Model

The individual tools solved individual problems. The collaboration model is what makes them compound instead of coexist—versioned skills that improve, shared data that flows between them, and quality gates that catch problems before they ship.

Skills That Learn

Early skills were useful once, then stale. A meta-skill now creates, evaluates, and iterates other skills on a 7-dimension framework—versioned releases, evaluation scores, learning logs. The resource cataloger went v1 → v3, interview-prep v0.3 → v0.5. Each version addresses gaps the previous one surfaced.

Skill evolution—version history across key skills showing feature progression and accuracy improvements over time

Interconnected Systems

Standalone tools create data silos. These skills share data through common hubs—the recruiter panel scores companies, outreach reads those scores to tailor messaging, interview-prep generates cheat sheets from the tracker and writes debrief notes back. One change propagates.

System map—data flow between skills, shared files, and living documents showing how one change propagates

Version Control and Quality Gates

Without automated checks, voice drift and broken markup ship silently. I set up git for every change and a CI/CD pipeline that runs voice-check, quality-check, and placeholder scan before anything deploys.

CI/CD pipeline—git push → voice check → quality check → placeholder scan → deploy

Portability by Design

Skills that only work in one environment aren’t reusable—they’re scripts. 6 of 9 skills are designed for sharing with parameterized paths, semver, and no hardcoded references. Portability scores tracked and improved across releases.

Portability audit—skills scored on shareability, showing path from internal tool to publishable artifact

04 Quality Over Time

The point of building systems instead of one-off deliverables is that quality compounds. Every revision runs through the same evaluation pipeline, so improvement is measurable and gaps don’t quietly reappear.

Measurable Improvement

Without a consistent scoring method, “better” is a guess. The recruiter panel scores every material against the same criteria each time—resume average moved from 6.7 → 8.2, case study scores climbed +0.5 to +1.1 per company after a single restructure pass.

Score progression—before/after across materials, showing measurable improvement per revision cycle

Voice Consistency

AI-assisted writing drifts toward generic patterns without guardrails. I defined 40+ banned patterns enforced by an automated linter, plus four voice registers so the system distinguishes between a LinkedIn post, a case study, and an outreach email.

Voice system—banned patterns, four registers, automated linting output

Results

The portfolio shipped in weeks. The lasting output is the practice infrastructure underneath it—9 interconnected skills, a shared tracker, a quality system scoring against live job postings, and a memory architecture that carries context across weeks of work.

Nothing here is static. New case studies get added, skills get new versions, the recruiter panel re-evaluates as postings change. The system is designed to keep compounding.
