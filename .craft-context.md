# Design Context — Della Sigrest Portfolio

## Direction
- **Feel:** Dark, sparse, confident — editorial not flashy
- **Audience:** Design hiring managers, recruiters, engineering leads at top-tier tech
- **Intent:** Prove staff-level systems thinking in 90 seconds; invite deeper reading

## Tokens
- **Primary:** #e0dfe4 (--text-primary)
- **Secondary:** rgba(224, 223, 228, 0.6) (--text-secondary)
- **Tertiary:** rgba(224, 223, 228, 0.4) (--text-tertiary)
- **Accent:** #7fb5b0 (--accent)
- **Accent hover:** #8ac4be (--accent-hover)
- **Background:** #0a0c16 (--bg)
- **Surface 1:** #10121c (--surface-1)
- **Surface 2:** #151722 (--surface-2)
- **Surface 3:** #1a1c28 (--surface-3)
- **Border subtle:** rgba(224, 223, 228, 0.04)
- **Border normal:** rgba(224, 223, 228, 0.07)
- **Border hover:** rgba(224, 223, 228, 0.10)

## Typography
- **Headings:** Inter, 600-700
- **Body:** Inter, 400-500
- **Mono:** JetBrains Mono, 400
- **Scale:** Base 16px, line-height 1.6

## Spacing
- **Base unit:** 4px increments (--spacing-xs: 4px through --spacing-3xl: 64px)
- **Side padding:** clamp(24px, 5vw, 64px)
- **Max width:** 1120px (--max-width)
- **Radius:** 12px (images), 8px (cards/placeholders)

## Patterns
- Case studies: hero with eyebrow → meta grid → body prose with h2 sections → image placeholders → results → next link
- Bold key phrases for scan speed
- Nav: fixed blur overlay
- Fade-in on scroll via IntersectionObserver

## Hard Rules (Non-Negotiable)
- **No orphans anywhere.** A single word alone on a line is never acceptable — headlines, body text, bullets, resume entries. Use non-breaking spaces (\u00A0 in JS, &nbsp; in HTML) to bind the last 2-3 words of a line together, or reword. Always verify by rendering.
- **Weight hierarchy must be visible.** Company-level entries must be visually distinct from team/project-level entries. In documents: use size differentiation (e.g., 10pt company vs 9pt team). In HTML: use font-weight, size, or color layering.

## Image Treatment
- Composite phone mockups at 75% max-width (desktop), 85% tablet, 95% mobile
- First image per case study gets hero sizing: 85% desktop, 90% tablet, 100% mobile
- Centered with 12px border-radius, fade-in on scroll
- Working files prefixed with `_` (excluded via .gitignore)

## Anti-patterns caught
- Some placeholders still present (strategy frameworks, metrics, AI case study)
- No interactive prototypes or embedded Figma frames
- Case studies all follow identical structure — no visual variety
