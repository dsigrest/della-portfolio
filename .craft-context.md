# Craft Context: Get-a-Job Portfolio (Della Sigrest)

**Project:** Portfolio site for Della Sigrest, IC4 Product Designer targeting Staff/Senior roles at Ramp, Anthropic, Meta, Figma, OpenAI, Cursor.

**Design Positioning:** Minimalist, systems-first, bold on information density. Every pixel earns its place. Type hierarchy and spacing do the heavy lifting — no decorative flourishes. Think Linear, Stripe, Figma dark mode aesthetic.

## Figma Variable Collections

1. **"Original Diagram"** (17 variables) — the warm cream palette from the original Sankey/data viz work. Surfaces, text, borders, and the original segment colors (red-orange core, gold casual, teal-green resurrected, blue new).

2. **"Portfolio"** (21 variables) — Della's portfolio design system, sourced from `styles.css`. Cool blue-black surfaces, warm off-white text, teal accent, and the portfolio segment palette.

Both share the same variable naming convention (`bg/canvas`, `surface/card`, `text/primary`, `segment/core`, etc.) so diagrams can be built against either system. Visual reference: see "Design System — Color Tokens" page in Figma.

## Typography System

| Level | Font | Weight | Size | Tracking | Use |
|-------|------|--------|------|----------|-----|
| Display | Inter/SF Pro | 700 | 56–96px | -0.03em | Page titles, hero statements |
| Heading | Inter/SF Pro | 600 | 32–48px | -0.02em | Section titles, card headlines |
| Subheading | Inter/SF Pro | 500 | 18–24px | -0.01em | Card subtitles, labels |
| Body | Inter/SF Pro | 400 | 16px | 0em | Paragraph text, descriptions |
| Caption | Inter/SF Pro | 400 | 14px | 0.05em | Metadata, dates, tags |
| Label | Inter/SF Pro | 500 | 11px | 0.08em | Category labels, badges (uppercase) |

**Rationale:** Weight contrast (700/600/500/400) creates scan hierarchy. Tracking opens up at the bottom (labels) and tightens at the top (display). All-caps labels with wide tracking create a visual texture distinct from sentences.

## Spacing Scale

- **Tight (within components):** 4px, 8px (icon-label gap, badge padding)
- **Comfortable (between components):** 16px, 24px (card spacing, form field gaps)
- **Generous (between sections):** 48px, 64px, 80px (section breaks, hero breathing room)

## Signature Micro-Interaction

**Staggered metric reveals:** When data viz metrics or case card outcomes appear, they fade + count-up on mount with 50ms stagger between elements (capped at 300ms). Creates intentionality, not auto-generated feel.

## Interaction Honesty

- Only `.href`, `[role="button"]`, `onclick` elements have hover effects
- No `cursor: pointer` on non-interactive elements
- Card hover (if card is a link): subtle background shift + border highlight, NO lift/shadow

## Design Principles

1. **Information density:** Dense layouts are fine if spacing and hierarchy prevent cognitive overload
2. **No decorative UI:** Every visual element serves function or critical information
3. **Type as primary design tool:** Bold weight contrasts, size jumps, tracking variation carry the design
4. **Minimal color:** One accent (#7fb5b0), everything else grayscale + accent
5. **Craft shows in detail:** Proper kerning pairs, weight hierarchy, micro-interactions that feel intentional

## Figma Output Conventions

When generating Figma frames via the Plugin API:

1. **Auto-layout everything editable.** Cards, text groups, stat rows — use auto-layout frames, not absolute positioning. The diagram canvas itself can be a regular frame for flow positioning, but all content within cards must be auto-layout so Della can edit text and have everything reflow.
2. **Group logically.** Flows in one group, node bars in one group, each card is its own auto-layout frame. Name groups clearly.
3. **Use stroked lines for flows.** Stroked vector paths are easier to edit than filled polygons — users can drag control points and adjust stroke weight directly.
4. **Name every layer.** Every element gets a descriptive name (not "Rectangle 47"). Flows: "Flow — [source] → [target]". Cards: "Card — [label]". Rows: "Row — [segment]".
5. **Segment rows = horizontal auto-layout.** Each row (dot + percentage + label) is a horizontal auto-layout frame with center alignment and 8px spacing.
